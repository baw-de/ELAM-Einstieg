# Contributing to ELAM-Einstieg

We welcome contributions to the **ELAM-Einstieg** project! By contributing, you help improve the model and ensure it is useful to the community. Please take a moment to review this document to understand the contribution process.

## How to Contribute

### Reporting Bugs
If you find a bug or have a suggestion for a feature, please open an issue on the GitHub repository. Make sure to provide enough details so that we can reproduce and address the problem. For example:
- The exact error message
- Steps to reproduce the bug
- Expected behavior vs. actual behavior

### Feature Requests
If you have an idea for a new feature or improvement, feel free to open an issue or contribute directly by submitting a pull request. Please explain the feature in detail and why it would be beneficial.

### Pull Requests
We encourage you to fork the repository and create a feature branch for your work. Afterward, submit a pull request with a clear description of what changes have been made and why. Please follow the steps below for submitting a pull request:

1. **Fork the repository**: Create your own copy of the project.
2. **Clone your fork**: Download the repository to your local machine.
3. **Create a branch**: Name your branch something descriptive (e.g., `feature-add-new-method` or `bugfix-fix-error`).
4. **Make changes**: Implement the bug fix or feature addition.
5. **Commit changes**: Write clear and concise commit messages that explain what and why you changed something.
6. **Push changes**: Push your changes to your forked repository.
7. **Submit a pull request**: Go to the original repository and open a pull request.

### Code Style
- Follow PEP 8 for Python code style.
- Use meaningful variable and function names.
- Add docstrings for functions, classes, and modules.
- Ensure your code is well-commented and readable.

### Testing
Before submitting your pull request, please ensure that the code passes all tests (if applicable).
